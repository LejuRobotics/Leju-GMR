# 1. S17 的bvh_to_csv脚本说明

## 使用示例
'''bash
python scripts/bvh_to_robot.py \
  --bvh_file "output/bvh/*.bvh" \
  --robot "biped_s17" \
  --save_path "output/qmai" \
  --rate_limit \
  --format "qmai"
'''

会直接在保存地址输出对应csv文件，目前机器人只支持s17，数据支持lafan1、qmai，保留参数方便添加

# 2. S17 千面视频上传即文件下载脚本使用说明

## 使用示例
'''bash
python scripts/qmai_video_to_bvh.py -f "*/video.mp4" -k "your key"
'''

## 请求参数使用说明
 -f | --file : 本地视频文件的路径(必填) \
 -u | --url : API 接口域名地址 (默认 `https://www.qmai.vip`) \
 -k | --key : 企业上传凭证，可在用户详情页查看（必填） \
 -c | --capture : 捕捉类型 (`capturetype`) (默认全身`0`) \
 -b | --bone ： 骨骼类型 (`bonetype`)（默认`15`，其他类型看骨骼类型详情表） \
 -p | --pose ： 第一帧姿态类型 (`posetype`) （默认TPose`1`） \
 -r | --frameRate : 输出帧率(计费相关): 24/30, 60, 120 (默认30帧，逗号分割代表不同价格) \
 -s | --standPose : 原地动作，默认关闭(开启后动作将在原地执行，可能对下肢还原后一定影响，不建议开启) \
 --physicType : 物理优化类型（计费相关）：1:(1.0版), 2:(2.0版) (默认1，0版) \
 --physicTimes : 迭代倍数，仅当 physicType=2 时有效，即使用2.0版物理优化。可选 1--6，越小动作越还原，越大动作越平滑。1x档：优先还原动作，足部稳定性可能较弱；2x-5x：兼顾还原和稳定性，倍率越高稳定性越好；10x：稳定性优先和锁足防滑，动作还原降低）\
 --rollback : 回调通知地址 (`rollbackurl`) (默认`http://192.168.141.68:80`) \

 -o | --output_dir_path : 输出文件目录地址(默认output/bvh) \
 --isDownload : 下载开关(默认打开) \
     
注意：isStaticCameraca(摄像机状态)， \
     mulPersonInfo(多人视频目标框信息，单人可不传), \ 
     physicTimes(仅当 physicType=2 时有效，可选 1--6（6代表10次）) \
     这三个参数未添加,如有需要可自行添加

## 使用建议
1. 按照官方“上传小贴士”规范视频质量 \
2. 疑似网页版已经全面使用物理优化2.0版模型(且价格降低至1cv1秒，需要证实)，2.0版物理优化模型追踪动作还原更好，稳定性更强 \
3. 按照需求选择pose type，capture type \
4. 其余变化可直接默认

动画基础模型1.0：通用需求，可以提供稳定可靠的基线动捕数据；推荐日常动作，舞蹈等常规动作。\
适用场景：\
室内平地面场景的角色行走、慢跑、原地转身、坐姿切换；\
无肢体遮挡、无剧烈重心变化的基础动作；\
短视频动画、简易游戏角色动作、基础三维演示片制作等。\


动画基础模型2.0(beta)：api暂不可用。官方描述中2.0在复杂动态动作捕捉上有提升，增强捕捉和运动预测。推荐再运动，攀爬，腾空，走跑，倒地等复杂动态上使用。\
不仅强化了复杂场景下的动作捕捉能力，尤其对运动、攀爬、滞空、走/跑、死亡/倒地等包含肢体发力、重心变化的复杂动态动作，实现了更细腻的姿态还原与轨迹追踪；\
更是具备了一定的动作预测能力，能够对镜头外的肢体动作进行推算，即使部分身体部位被遮挡或超出画面范围，也能输出自然的动作数据。\
无论是户外复杂地形，还是室内交互场景，都能大幅提升复杂动作的制作效率与真实感。\

## 一些参数表
capturetype: 动捕类型，多个值用逗号分隔，如 "0,2,3" \
    0：全身（默认） \
    1：半身 \
    2：手捕 \
    3：面捕 \
    5：自动判断（全身/半身） \
注意：全身/半身/自动判断只能选一个 \

posetype: 第一帧姿态类型 \
    1：TPose \
    2：APose \
    3：原Pose \
